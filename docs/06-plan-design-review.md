# Design Review: 等我有钱了

## 设计判定

这个产品的设计核心不是“豪华 UI”，而是“可传播表达格式”。

如果界面很贵气，但用户不愿意分享，那就是失败。

## 10 分设计标准

10 分版本应该满足：

- 标题一眼懂：等我有钱了。
- 输入句式自然：等我有钱了，我要...
- 输出内容像朋友会转发的图文，不像系统通知。
- 分享卡能独立存在，即使对方没装 App 也能看懂。
- 气质是轻喜剧、愿望、反差，不是贷款广告、炫富海报或低质测试。

## 视觉方向

不建议第一版做重奢视觉。

原因：

- 重奢容易变成假、高冷、炫富。
- 用户传播时会有心理负担。
- 审核上也更容易被误解为金融或消费诱导。

建议方向：

- 温暖、松弛、带一点荒诞。
- 像“人生愿望便利贴 + 富人段子日报”。
- 用真实生活愿望做入口，不用豪车豪宅做入口。

## 分享卡原则

分享卡必须包含：

- 用户原句
- 生成结果
- 一个富人烦恼或反差梗
- “虚拟娱乐模拟”轻提示

不要包含：

- 银行卡 UI
- 真实支付成功
- 真实余额
- 贷款额度
- 投资收益
- 充值按钮

## 文案语气

推荐：

- “等我有钱了，我要把周一改成私人假期。”
- “以前烦地铁太挤，现在烦私人飞机没有靠窗位。”
- “等我有钱了，我先给自己买一段不用解释的休息。”

避免：

- “马上暴富”
- “财富自由方案”
- “黑卡额度已到账”
- “充值获得更多现金”
- “你的资产超过 99% 的人”

## 设计结论

第一版不要从 App UI 开始。

先做 3 类可分享内容模板：

1. 愿望实现卡
2. 富人烦恼卡
3. 自我奖励卡

模板验证通过后再设计 H5 或 App。
