# 产品决策总账：等我有钱了

状态：当前有效  
更新日期：2026-07-02  
优先级：本文件高于 `docs/03-office-hours.md`、`docs/04-plan-ceo-review.md`、`docs/05-content-validation-plan.md` 中的早期方向判断。早期文档保留为思考过程，本文件用于后续校对、取舍和产品打磨。

## 一句话定义

“等我有钱了”不是虚拟购物 App，也不是聊天 App。

当前定义：

> 一个私人管家式有钱爽剧生成器。用户负责想爽，系统负责懂爽。

用户输入一句“等我有钱了，我要...”，或者直接选择“管家来安排”，系统把愿望安排成一段短、准、具体、有递进、有高潮、可分享的有钱爽剧。

核心不是让用户拥有虚拟余额，而是让用户短暂进入一个世界：

> 一觉醒来，世界已经默认你很有钱。

## 最高原则

产品要服务的不是“钱”，而是“被有钱世界认真对待”的体感。

必须持续守住三件事：

1. 不做违法、伤害、赌博、色情、真实金融误导。
2. 不冷冰冰拒绝，遇到风险愿望用管家式转译。
3. 每个幻想都落到具体服务细节，但不让用户为小事做决定。

最重要的一条体验原则：

> 老板不能累。

如果系统把用户带进有钱生活，却让用户不停选择、确认、配置、判断，那就把爽感做反了。真正的管家式服务不是“要不要为您安排”，而是“已经为您安排好，您只需要出现”。

## 我们不做什么

第一阶段明确不做：

- 不做真实金融、贷款、理财、投资、额度、提现、返利。
- 不做博彩、下注、筹码、赔率、赢钱输钱。
- 不做色情、擦边、身体交易或低俗服侍。
- 不做虚拟黑卡余额、虚拟现金包、充值买金币。
- 不做全球花钱排行榜或强攀比榜。
- 不做复杂社交网络。
- 不把现有 Expo 原型继续堆成主线产品。
- 不在内容验证不足时直接冲 App Store 或国内安卓市场。

这不是因为这些功能都不能想，而是它们会把产品从“情绪价值娱乐”拉向金融、博彩、低俗炫富或高成本工程。

## GStack 决策路径

### Q1：这个需求真的存在吗？

初始信号成立，但还没有完全证明。

信号来自：

- 大家日常会说“等我有钱了，我要...”，而且越聊越起劲。
- 郭德纲相声里也有类似“等我有钱了”的荒诞富豪幻想。
- 微信群、朋友、家人、同事聊天里，经常用这句话接梗。
- 小红书、抖音、豪宅、豪车、酒店、探店、机票、奢侈品内容，本质上都在提供代偿式想象。
- 韩国虚拟购物网站的走红说明，纯模拟消费也可以提供情绪价值。

判断：值得继续验证，但不能直接开发完整 App。

### Q2：用户现在怎么解决这个需求？

用户现在的替代行为包括：

- 在群聊里说“等我有钱了...”然后互相接梗。
- 刷豪宅、豪车、酒店、会所、奢侈品探店内容。
- 逛淘宝、得物、携程、高德酒店、机票、豪华体验，但不下单。
- 收藏链接、加购物车、看价格、做幻想清单。
- 玩身份测试、人生重开、财富测试、MBTI 类小游戏。
- 发朋友圈、微博、小红书时把“等我有钱了”当段子。

当前替代方案的问题是：都停留在想象，没有让用户真正进入“已经有钱”的世界。

更准确的需求不是“我要模拟花钱”，而是：

> 我想短暂进入一个有钱世界，并且这个世界认真对待我。

### Q3：第一批种子用户是谁？

目前最清晰的种子用户画像：

- 30-45 岁，有现实压力，会用玩笑表达欲望的人。
- 会在群里接梗，愿意把现实压力变成段子的人。
- 会刷豪宅、豪车、酒店、探店、奢华服务内容的人。
- 想象过“等我有钱了我要...”的人。

已讨论样例：

> 40 岁，男，互联网 17 年。愿望是当富豪，买上海汤臣一品，阳台看到东方明珠，豪车十多辆，吃饭十几个人服务。

这类用户的爽点不是简单“买贵东西”，而是身份反转：现实里要解释、等待、排队、妥协；幻想里别人已经认识他、等他、围着他安排。

### Q4：最小可验证产品是什么？

当前最小闭环不是 App，也不是完整剧情系统。

最小可验证产品是：

> 60 秒内看完的一段私人管家式有钱爽剧首秀。

两个入口：

1. 我有一个愿望：用户输入“等我有钱了，我要...”
2. 管家来安排：用户不知道要什么，系统替他安排今日有钱人生。

输出必须包含：

- 今日身份。
- 管家开场。
- 有递进的短剧分镜。
- 一个具体到位的服务细节。
- 今日富豪战报，适合截图或转发。

### Q5：第一轮人工测试学到了什么？

已做一轮真人手动测试，记录在 `docs/10-manual-q5-test.md`。

用户 A 原始愿望：

> 等我有钱了，我要去最高档的会所，体验美女如云和众星捧月、人上人的快感。

三轮反馈：

- 第一版：4 分。问题是太短、部分句子不通顺、卡片不可见。
- 第二版：3.5 分。问题是细节增加了，但不丝滑，没有层层递进到高潮。
- 第三版：4.5 分。短剧分镜明显提升爽感。

关键学习：

- “美女如云”不能直接删除，否则会把原始爽点洗掉。
- 正确做法是保留“赏心悦目、被欣赏、被围绕”的情绪内核，转译为体面、专业、克制的高端社交场。
- 爽感要递进，不要平铺卡片。
- 不能让老板做小决定。
- 细节要服务高潮，不要堆奢侈名词。

当前判断：方向有明显正反馈，但样本不足。还不能进入工程开发。

## 目标爽感

产品最终要让用户感到：

- 我被识别了。
- 我被优先了。
- 我不用解释。
- 我不用排队。
- 我不用做小决定。
- 有人记得我的偏好。
- 有人在我开口前已经安排好。
- 旁人看到了我被认真对待。
- 我一句随口的话，现场规则就变了。
- 最后还有一句可以发出去的战报。

强爽点不是“我花了很多钱”，而是：

> 我一出现，世界开始按我的节奏运转。

## 第一体验路径

### 首次进入

1. 用户看到两个入口：
   - 我有一个愿望
   - 管家来安排
2. 用户输入愿望，或让管家自动安排。
3. 用户可输入昵称、别人如何称呼他、虚拟形象。照片上传不做 v1。
4. 黑屏短暂过渡：

```text
正在重写您的生活设定...
```

5. 用户从“睡梦中醒来”，管家开场：

```text
老板，您终于醒了。
昨晚您说想住在能看见东方明珠的地方，所以管家已经把阳台的风调小了。
东方明珠今晚会亮得克制一点，免得打扰您喝茶。
```

6. 进入短剧分镜。
7. 结束时生成今日富豪战报。

### 后续打开

不重复完整设定。

后续打开应是：

```text
老板，您回来了。
昨天您随口说过的事，今天已经有结果了。
```

后续可给三个方向：

- 继续昨天的安排。
- 今日管家来安排。
- 换一种爽法。

## 两种核心入口

### 入口 A：愿望模式

用户输入：

```text
等我有钱了，我要住能看见东方明珠的房子。
```

系统任务：

- 不判断愿望大不大。
- 不要求愿望现实。
- 不把愿望做小。
- 把它认真安排成具体场景。

用户输入“我要去月球”也可以。原则是：

> 不限制愿望大小，限制表达方式。

### 入口 B：管家来安排

用户不输入愿望，只表达一种状态：

```text
我今天累了，管家来安排。
```

系统任务：

- 主动决定今日身份。
- 主动安排场景。
- 主动给出服务细节。
- 主动给出可分享战报。

这一入口很重要，因为很多用户并不知道自己要什么。他们真正想要的是：我已经有钱了，你是专业的，你来迎合我，我只管享受。

## 生成内容结构

后续 AI 生成不要只输出“三张卡”。

当前推荐结构：

1. 风险转译：把危险或低俗表达转成安全但保留爽点的表达。
2. 爽点内核：识别用户真正想体验的情绪。
3. 短剧分镜：入场、优先、见证、改规则、高潮。
4. 富豪战报：压缩成可分享结果。

短剧分镜建议：

1. 入场：用户被识别。
2. 优先：原本服务别人的人开始围着用户转。
3. 见证：朋友或旁人看到后震惊。
4. 改规则：用户一句话改变现场安排。
5. 高潮：主理人亲自出面，今晚按用户标准来。
6. 战报：今日成就、朋友反应、管家评语。

卡片可以作为 UI 形式，但内容必须有剧情推进。

## AI Agent 设定

未来如果用 Agent 化生成，不应让一个大模型随便发挥。

建议拆成一组“专业管家团队”：

| Agent | 职责 |
| --- | --- |
| 愿望解析 Agent | 从用户输入里识别真实爽点：自由、服务、面子、休息、补偿、荒诞、被看见 |
| 风险转译 Agent | 处理色情、赌博、违法、金融误导等高风险表达，转译但不洗掉欲望 |
| 管家设定 Agent | 决定称呼、今日身份、管家语气、开场仪式 |
| 服务细节 Agent | 给出 1-2 个具体但不繁琐的高价值细节 |
| 爽剧分镜 Agent | 生成递进剧情，保证有入场、优先、见证、改规则、高潮 |
| 分享战报 Agent | 把结果压缩成可截图、可转发、可被朋友接梗的战报 |

核心约束：

> AI 不是写奢侈品清单，而是替用户安排一个“被认真服务”的世界。

## 内容语言规则

### 保留欲望，不洗掉欲望

错误：

```text
用户说“美女如云”，系统直接改成“专业服务团队”。
```

问题：安全了，但爽点没了。

正确：

```text
今晚接待您的不是热闹，是审美。经理带来的团队都很会把距离拿捏好：看得见您的存在，也懂得不打扰您的兴致。
```

原则：

> 安全不是删除欲望，而是把欲望升级成体面、审美和情绪价值。

### 用动作替代抽象大词

弱句：

```text
您不是来消费的，您是来让这个场子成立的。
```

改法：

```text
您刚坐下，主理人就取消了原流程。灯光降一档，音乐慢下来，服务线往后退半步。今晚不是按菜单走，是按您舒服的节奏走。
```

### 细节要少，但要准

细节不是为了显得高级，而是为了证明世界在认真对待用户。

好细节：

- 杯壁薄半毫米，因为老板第一口不该被玻璃打断。
- 灯光提前降一档，因为老板还没开口，现场已经进入他的状态。
- 纽扣换成贝母扣，因为近距离时，别人会安静多看一秒。

坏细节：

- 一口气堆名表、名车、豪宅、香槟、黑卡、游艇。
- 每个细节都像奢侈品广告。
- 细节很多，但没有剧情推进。

## 风险转译规则

### 色情或擦边

允许：

- 赏心悦目的服务团队。
- 气质出众的接待。
- 被欣赏、被温柔称赞。
- 体面、克制、有审美的高端社交场。

不允许：

- 色情暗示。
- 身体交易。
- 擦边挑逗。
- 低俗服侍。
- 把女性或任何人当成奖品。

### 赌博

不做：

- 下注。
- 筹码。
- 赔率。
- 输赢。
- 赌场玩法。

可转译：

- 澳门夜游。
- 高端酒店、包厢、表演、车队、服务线。
- “纸醉金迷但不下注”的场景体验。

### 真实金融误导

不做：

- 余额到账。
- 额度开通。
- 提现成功。
- 收益率。
- 投资回报。
- 黑卡真实申请。

可转译：

- 今日身份。
- 今日安排。
- 虚拟娱乐模拟。
- 管家档期。
- 私人订制券。

## 留存与增长判断

早期不要把次日留存当第一指标。

这个产品的第一指标是：

> 用户第一次爽完，会不会想截图、发群、让朋友也来一次。

第二天不来没关系，第一天不分享就危险。

增长闭环应是：

1. 用户生成自己的有钱爽剧。
2. 得到一张“今日富豪战报”。
3. 分享给朋友。
4. 朋友点击后看到“某某老板今日战报”，并生成自己的愿望。
5. 原用户收到朋友回应或接梗。

不要早期做重社交。先验证内容能不能让人想发。

## 成本与收费判断

AI 成本必须从第一天进入产品设计。

不建议：

- 一次性付费下载。因为 AI 成本是持续的。
- 纯无限包月。重度用户会把成本打穿。
- 卖虚拟钱、黑卡余额、金币。容易金融化或游戏化。

建议：

- 免费：每日 1 次普通演绎。
- 分享奖励：获得普通演绎次数。
- 付费：高级演绎次数、私人订制券、高清战报。
- 会员：固定额度的高级演绎、管家记忆、专属称呼、优先生成。

可用命名：

- 高级演绎。
- 管家档期。
- 私人订制券。
- 爽剧次数。
- 今日安排。
- 高清战报。

成本控制原则：

- v1 文本优先，图片后置。
- 视频不做 v1。
- 一次 AI 调用尽量生成完整结构。
- 小模型做分类和风险，大模型做最终文案。
- 免费用户不能无限生成。

## 技术方向

纯客户端不适合最终形态，因为：

- 云端 AI Key 不能安全放在客户端。
- 分享链路需要服务端。
- 用户从朋友圈点进来需要 H5 落地页。
- 需要匿名事件追踪判断分享和转化。

允许服务端，但服务端要薄。

第一阶段服务端只承担：

- AI 代理。
- 分享链接 `/s/xxx`。
- 匿名事件记录。
- 轻量用户设定和故事状态。

不做：

- 复杂账户系统。
- 资产系统。
- 全球排行榜。
- 重社交关系链。
- 金融账单模型。

## 当前建设门禁

在进入 H5 或 App 实现前，必须先满足：

- 至少完成 3-5 个真人手动测试。
- 至少 2 个用户评分 4 分以上。
- 至少 1 个用户主动想再来一个。
- 至少 1 个用户表示愿意截图、发群或给朋友玩。
- 至少验证 1 种分享战报格式能被朋友理解。
- 风险转译规则能处理色情、赌博、金融误导三类危险输入。

满足后，进入：

> H5 + 薄服务端 + AI 代理的可分享原型。

仍不建议第一步做完整 App。

## 后续决策检查表

任何新功能进入前，先问：

1. 它是否增强“被认真服务”的爽感？
2. 它是否让老板更轻松，而不是更累？
3. 它是否保留原始欲望，而不是洗掉欲望？
4. 它是否有递进和高潮，而不是卡片平铺？
5. 它是否能被截图、转发、接梗？
6. 它是否避开金融、赌博、色情、违法和低俗炫富？
7. 它是否能控制 AI 成本？
8. 它是否可以先人工测试，再工程实现？

如果答案有 3 个以上是否定，就不要开发。

## 未解决问题

下一轮重点验证：

- 这个模式是否只适合男性“霸总爽剧”，还是也能适合女性用户、自我奖励、被照顾、休息补偿。
- “管家来安排”是否比“我有一个愿望”更容易启动。
- 哪种战报最适合分享：称号、成就、朋友反应、管家评语、短剧截图。
- 用户是否愿意为“高级演绎”付费。
- 朋友点击分享后，最自然的下一步是接梗、评分，还是生成自己的愿望。

## 当前结论

方向不再是“虚拟富豪消费模拟器”。

当前方向是：

> 私人管家式有钱爽剧。

第一阶段不急着开发。先继续手动验证内容和分享。只有当真人测试证明“爽、想继续、想分享”成立，才进入 H5/server-backed prototype。
