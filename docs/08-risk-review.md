# Risk Review: 等我有钱了

## 风险判定

项目可以做，但必须保持“娱乐表达工具”的定位。

最大风险不是技术，而是被平台、用户或市场理解成：

- 金融产品
- 贷款产品
- 博彩/抽奖产品
- 低俗炫富工具
- 诱导现实消费的工具

## 高风险表达

禁止使用：

- 真实余额
- 额度到账
- 黑卡申请
- 提现
- 返利
- 投资收益
- 贷款
- 躺赚
- 资产超过多少人
- 充值获得现金

## 高风险机制

第一阶段禁止：

- 充值买虚拟现金
- 随机抽奖
- 抽卡
- 下注
- 开箱
- 排行榜攀比
- 真实品牌奢侈品误导
- 模拟真实银行账单

## 安全表达

推荐使用：

- 愿望
- 想象
- 虚拟人生
- 娱乐模拟
- 分享文案
- 荒诞烦恼
- 自我奖励

必须保留：

> 所有资产、身份、金额和消费均为虚拟娱乐模拟，不代表真实财富、金融服务、贷款额度或投资建议。

## 上架定位

如果未来进入 App：

- 类别优先：娱乐 / 生活方式
- 避免类别：游戏、金融、博彩、理财
- 文案避免：“黑卡”“额度”“现金流”等金融强联想作为主卖点

国内安卓尤其要避免被识别成网络游戏，否则可能触发版号、防沉迷、实名等高门槛。

## 用户心理风险

需要避免：

- 强化阶层焦虑
- 羞辱普通生活
- 把真实贫穷当笑料
- 鼓励现实高消费
- 鼓励借贷消费

建议气质：

- 自嘲
- 轻喜剧
- 愿望表达
- 对现实压力的温柔转译

## 风险结论

通过，但有条件：

1. 第一阶段只做内容验证。
2. 不做虚拟钱包/黑卡/账户系统。
3. 不做充值或随机奖励。
4. 分享卡必须标明虚拟娱乐模拟。
5. 产品主语从“我多有钱”改成“我的愿望被荒诞地实现了”。
